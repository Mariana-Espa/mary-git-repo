Git & GitHub

Que es GitHub?

GitHub es un sitio web y un servicio en la nube que ayuda a los desarrolladores a almacenar y administrar su código, llevar un registro y control de cualquier cambio sobre este código.

los sistemas de control de versiones: nos ayudan a nuestro flujo de desarrollo, a mantener un historial de cambios; a tener control sobre cada cambio en el código; a que un cambio de una persona no influya en el cambio realizado por otra;

Permite el almacenamiento y acceso a un historial de cambios

Configuraciones

git config --local: nos indica que las configuracioones que vamos a crear van a ser solo para este proyecto

git config --global: sirve para todos los proyectos que haremos desde ese momento en la computadora

git config --global user.email "correo@gmail.com": permite configurar o agregar tu email

git config --global user.nombre "nombre": permite configurar tu nombre

Algunos comandos de GitHub

cd: nos permite navegar por nuestro directorio

ls: nos muestra los archivos y documentos de donde nos encontramos

git --version: nos muestra la version instalada

git init: inicializa un repositorio en el directorio donde se ejecuta el comando. Desde este comando, Git podrá administrar los cambios realizados en los archivos.

Cuando git init veremos que nos aparece la palabra master, esto significa que esta en la branch master

git status: nos muestra los cambios y el estado de los archivos

Al ejecutar el comando git status, recibimos información que puede no ser tan clara, especialmente cuando nos encontramos con términos como HEAD, working tree, index, etc.

  • head: Estado actual de nuestro código, es decir, donde nos colocó Git
  • Working tree: Lugar donde los archivos realmente están siendo almacenados
  • index: Lugar donde Git almacena lo que será commiteado, es decir, la ubicación entre el working tree y el repositorio de Git en sí.

git add: monitorea un archivo

git add . : monitorea todos los archivos de la carpeta donde nos encontramos

git rm --cached "nombre del archivo": deja de monitorear el archivo seleccionado

git commit -m "descripcion": guardamos la version de codigo

un commit es la forma de guardar un estado o versión de nuestro código;

.gitignore: permite ignorar un archivo de nuestro proyecto

Git log

git log: muestra el hitorico de modificaciones

git log --oneline: nos muestra un resumen de los cambios realizados

git log -p: nos muestras todas las modificaciones realiozadas
